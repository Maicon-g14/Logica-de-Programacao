Neste laboratório vamos criar um programa para gerenciar um repositório de dados (base de dados) com registros acadêmicos de alunos. Cada aluno possui os seguintes dados:  RA, e-mail, nome, usuário e senha. As operações que poderão ser realizadas sobre o repositório são i) adição de um aluno, ii) alteração de um aluno, iii) impressão de um aluno, e iv) remoção de um aluno.

O repositório será implementado na classe Repositorio (arquivo lab17.py), que contém os métodos referentes as operações mencionadas anteriormente e um atributo alunos. O atributo alunos representa a lista de alunos cadastrados no repositório.

Um registro com as informações de um aluno é uma instância da classe Aluno (arquivo aluno.py):

class Aluno:
    def __init__(self, nome, ra, email, usuario, senha):
        self.nome = nome
        self.ra = ra
        self.email = email
        self.usuario = usuario
        self.senha = senha

Esse laboratório possui 3 arquivos: lab17_main.py, aluno.py, lab17.py. O arquivo lab17.py possui 3 exceções: EmailInvalido, SenhaFraca e RAInvalido. Você deverá implementar apenas a classe Repositorio do arquivo lab17.py. Os métodos da classe Repositorio são:

  • limparRepositorio: Este método exclui todos os alunos do repositório.

  • adicionar: Este método recebe o parâmetro aluno e o adiciona no repositório, caso não tenha nenhum problema. Existem 3 situações nas quais essa operação falha:

    1. Caso outro aluno com o mesmo RA já exista no repositório. Nesse caso, o aluno não é inserido no banco de dados e o método deve lançar a exceção RAInvalido.
    2. Caso a senha fornecida não seja forte. Para que uma senha seja considerada forte ela tem que conter ao menos 1 letra maiúscula, ao menos 2 letras minúsculas, ao menos 2 números, ao menos um caracter especial (!@#$&*) e ter um tamanho maior ou igual a 8. Se a senha fornecida não for forte, o aluno não é inserido e o método deve lançar a exceção a frase SenhaFraca.
    3. Caso o email fornecido esteja em um formato inválido. O formato aceito para emails é usuario@servidor.dominio. O usuario pode conter letras, números e os seguintes caracteres: _ . + -. O servidor só pode conter letras e números. O dominio só pode conter letras e só deve conter de 2 a 4 caracteres. Se o email fornecido não estiver no padrão aceito, o aluno não é inserido no banco de dados e o método deve lançar a exceção EmailInvalido.

    Caso não haja problemas com os dados fornecidos, o novo aluno deve ser adicionado e a seguinte mensagem é impressa pelo programa principal Adicionado: RA - E-MAIL - NOME - USUARIO, onde RA, E-MAIL, NOME e USUARIO são substituídos pelos dados do novo aluno.

  • alterar: Este método recebe o parâmetro aluno e altera no repositório, caso não tenha nenhum problema, os dados do aluno com RA igual a aluno.ra. Existem 3 situações nas quais essa operação falha:

    1. Caso nenhum aluno do repositório tenha RA igual a aluno.ra, a operação falha e o método deve lançar a exceção RAInvalido.
    2. Caso a senha fornecida não seja forte. Para que uma senha seja considerada forte ela tem que conter ao menos 1 letra maiúscula, ao menos 2 letras minúsculas, ao menos 2 números, ao menos um caracter especial (!@#$&*) e ter um tamanho maior ou igual a 8. Se a senha fornecida não for forte, o aluno não é alterado e o método deve lançar a exceção SenhaFraca.
    3. Caso o email fornecido esteja em um formato inválido. O formato aceito para emails é usuario@servidor.dominio. O usuario pode conter letras, números e os seguintes caracteres: _ . + -. O servidor só pode conter letras e números. O dominio só pode conter letras e só deve conter de 2 a 4 caracteres. Se o email fornecido não estiver no padrão aceito, o aluno não é alterado no banco de dados e o método deve lançar a exceção EmailInvalido.

    Caso não haja problemas, o aluno deve ser alterado e a seguinte mensagem é impressa pelo programa principal Alterado: RA - E-MAIL - NOME - USUARIO, onde RA, E-MAIL, NOME e USUARIO são substituídos pelos dados do parâmetro aluno.

  • achaAluno: Este método recebe o parâmetro ra e deve retornar o aluno com aluno.ra igual ao parâmetro. Se o aluno não estiver cadastrado, o método deve lançar a exceção RAInvalido. Caso não haja problemas, o programa principal imprime uma mensagem no formato RA - E-MAIL - NOME - USUARIO.

  • remover: Este método recebe o parâmetro ra e deve remover o aluno correspondente do repositório. Se o aluno não estiver cadastrado, o método deve lançar a exceção RAInvalido. Caso não haja problemas, o programa principal imprime a mensagem Aluno RA removido com sucesso, onde no lugar de RA é impresso o RA passado como parâmetro.

Reforçando

Neste laboratório você não precisará se preocupar em ler a entrada a partir da entrada padrão. Seu trabalho é apenas implementar os métodos da classe Repositorio.

As sessões abaixo, de Entrada e Saída, descrevem os formatos de entrada e saída, mas você não precisa se preocupar com o formato da Entrada.

Entrada

A primeira linha da entrada contém um inteiro n indicando o número máximo de alunos que a base conterá.

As linhas seguintes consistem de operações a serem realizadas na base de alunos. As operações são:

  • > RA Chama o método achaAluno passando o RA informado;
  • + RA E-MAIL USUARIO SENHA NOME Adicionar um aluno com os dados fornecidos. Caso o email esteja em um formato errado, imprime uma mensagem de erro;
  • * RA E-MAIL USUARIO SENHA NOME Alterar um aluno com os dados fornecidos. Caso o email fornecido esteja em um formato errado, imprime uma mensagem de erro;
  • - RA Remover o aluno de RA informado do repositório;
  • q Limpar a base e encerrar o programa.

Onde:

  • RA é o número do RA;
  • E-MAIL é o e-mail do aluno;
  • NOME é o nome do aluno;
  • USUARIO é o usuário do aluno.
  • SENHA é a senha do aluno;

Saída

Cada linha da saída do programa contém o resultado da execução de cada operação dada na entrada, de forma que a saída possui uma linha a menos que a quantidade de linhas da entrada.
